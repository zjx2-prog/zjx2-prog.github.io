nacos基本使用，不讲理论

（1）nacos下载和安装（Windows安装）

#nacos下载网址，若下载速度慢，请翻墙
https://github.com/alibaba/nacos/releases

#下载下来的是一个压缩包，解压即可使用
  • 启动前配置（默认以集群方式启动，现在修改为单机模式）

  • 最后双击startup.cmd启动

测试nacos是否可以使用

  • 使用curl工具测试nacos的open api。

    • 下载curl
    #下载地址
https://curl.haxx.se/windows/
    • 发布配置
    #1.下载完成后进入curl的bin目录
#2.并在当前文件夹下打开cmd或者打开cmd后进入到bin目录也可以
#3.然后输入先这串代码进行测试

curl -X POST "http://127.0.0.1:8848/nacos/v1/cs/configs?dataId=nacos.cfg.dataId&group=test&content=HelloWorld"

#这个命令表示想nacos发布一个配置，运行命令完成后就可以在nacos中的配置列表中看到两个配置文件
    • 获取配置
    #向nacos发布配置成功，就可以通过客户端从nacos获取配置信息，执行以下命令
curl -X GET "http://127.0.0.1:8848/nacos/v1/cs/configs?dataId=nacos.cfg.dataId&group=test"

（2）nacos配置管理

nacos配置管理-快速入门

  • 在nacos中新建一个配置文档

  • 通过java获取配置文档里的信息
<!--新建一个工程，然后在pom文件中导入客户端的依赖-->
<dependency>
     <groupId>com.alibaba.nacos</groupId>
     <artifactId>nacos-client</artifactId>
     <version>2.2.1</version>
 </dependency>
  • 编写一个测试类
public static void main(String[] args) {

        Properties properties = new Properties();
        properties.put("serverAddr","localhost:8848");
        properties.put("namespace","77d71910872d6e7907846a52275f0ed0");

        try {
            ConfigService service = NacosFactory.createConfigService(properties);
            String s = service.getConfig("nacos-demo.yml", "DEFAULT_GROUP", 5000);
            System.out.println(s);
        } catch (NacosException e) {
            throw new RuntimeException(e);
        }
    }

​ 以上便是nacos的配置管理的基本使用。

配置管理-命名空间

  • 进入nacos后，点击左侧菜单栏中的命名空间。

  • 点击右上角的新建命名空间，输入名称和描述即可。

配置管理-功能操作

  • 对于基本的查看详请、删除、修改这些功能这里不再过多讲解。这里主要讲解一下历史版本的回滚操作。

监听查询

 nacos提供了监听的功能，以便客户端能够第一时间接收到服务端的变化。
//在测试类中加入以下代码	
		  String dataId = "nacos-demo.yml";

        String group = "DEFAULT_GROUP";

        Properties properties = new Properties();
        properties.put("serverAddr","localhost:8848");
        properties.put("namespace","edc2671d-5f70-435a-8c14-62ecb095a2ce");

        try {
            ConfigService service = NacosFactory.createConfigService(properties);
            String s = service.getConfig( dataId, group, 5000);
            System.out.println(s);

            service.addListener(dataId, group, new Listener() {
                @Override
                public Executor getExecutor() {
                    return null;
                }

                @Override
                public void receiveConfigInfo(String configInfo) {
                    System.out.println(configInfo);
                }
            });
        } catch (NacosException e) {
            throw new RuntimeException(e);
        }

        while(true){
            try {
                Thread.sleep(2000);
            } catch (InterruptedException e) {
                throw new RuntimeException(e);
            }
        }

  • 最后去配置列表中修改demo中定好的配置文件的内容，便可在控制台中看到修改过后的配置文档的内容。上面的demo指定的是nacos-demo.yml这个配置文件。

自定义扩展dataId

我们可以使用extension-config来共享多个不同组的配置文件

spring:
  application:
    name: nacos-mysql2
  cloud:
    nacos:
      config:
        server-addr: localhost:8848
        file-extension: yml
        namespace: e829ab19-8336-453b-ba13-4dca39b63897
        group: DEFAULT_GROUP
        extension-configs[0]: #获取data-id为nacos-mysql.yml的配置文件
          data-id: nacos-mysql.yml
        extension-configs[1]: #获取data-id为nacos-mysql.yml的配置文件
          data-id: nacos-mysql3.yml

创建客户端服务

  • 新建一个子模块nacos-consumer，添加以下依赖
<dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>
        <dependency>
            <groupId>com.alibaba.cloud</groupId>
            <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
            <version>2021.0.4.0</version>
        </dependency>
	<!--添加feign远程调用依赖-->
        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-starter-openfeign</artifactId>
            <version>3.1.5</version>
        </dependency>
	<!--高版本的springcloud需要添加这个依赖，不然会报错-->
        <dependency>
            <groupId>org.springframework.cloud</groupId>
            <artifactId>spring-cloud-starter-loadbalancer</artifactId>
            <version>3.1.5</version>
        </dependency>
  • 添加配置
server:
  port: 8010
spring:
  application:
    name: nacos-consumer
  cloud:
    nacos:
      discovery:
        server-addr: localhost:8848   #服务发现中心的地址
  • 声明Feign客户端
package com.zjx.nacosConsumer.client;

import org.springframework.cloud.openfeign.FeignClient;
import org.springframework.web.bind.annotation.GetMapping;

@FeignClient(value = "nacosServers")
public interface ProviderClient {

    @GetMapping("/tt")
    public String tt();//调用生产者服务端controller层的tt方法

}
  • 创建一个controller，用来待会浏览器请求调用。
@RestController
public class ConsumerController {

    @Resource
    private ProviderClient providerClient;
    @GetMapping("/pp")
    public String ttttt(){
        System.out.println("远程调用");
        return providerClient.tt();
    }
}
  • 启动类添加Feign注解
@SpringBootApplication
@EnableDiscoveryClient
@EnableFeignClients
public class nacosConsumerApplication {
    public static void main(String[] args) {
        SpringApplication.run(nacosConsumerApplication.class,args);
    }
}

以上，客户端服务创建完成。

服务注册选择

在nacos中我们可以看到服务列表也会有命名空间的区分。为此我们可以选择将服务注册到其他的命名空间。在配置文旦中添加namespace，便可将服务注册到指定的命名空间。

server:
  port: 8010
spring:
  application:
    name: nacos-consumer
  cloud:
    nacos:
      discovery:
        server-addr: localhost:8848   #服务发现中心的地址
        namespace: 2230bc7a-2d7c-46ab-917e-b68462964dbc  #将服务注册到另外一个命名空间
        cluster-name: DEFAULT   #指定集群的名称，集群名称默认DEFAULT
